---
name: Zig 0.14.1 std.http.Client API
description: what changed vs older Zig http client code (Headers struct removed) and the correct low-level call shape
---

Zig 0.14.1's std.http.Client removed the old `http.Headers` struct and the `client.open(method, uri, headers, options)`
4-arg signature. Code written against older Zig (or generated by an LLM from stale training data) will fail with
"root source file struct 'http' has no member named 'Headers'".

**Why:** upstream Zig std.http API churn between 0.10-ish and 0.14 releases changed low-level header handling.

**How to apply:** use `client.open(method, uri, .{ .server_header_buffer = &buf, .extra_headers = &[_]std.http.Header{...} })`
(3-arg form, headers passed inside the options struct). `req.send()`/`req.finish()`/`req.wait()` take no arguments.
See a working example already in this codebase's src/core_relational/quantum_hardware.zig before writing new
std.http.Client call sites.
